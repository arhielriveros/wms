## Nivel 3 — Componentes del backend

```mermaid
flowchart LR
  Host["API/Worker Host"] --> Platform["Platform + Tenancy"]
  Host --> Security["Security + Audit"]
  Host --> Layout["Layout"]
  Host --> Master["Master Data"]
  Host --> Inbound["Inbound"]
  Host --> Outbound["Outbound"]
  Host --> Mobile["Mobile Sync"]
  Inbound --> Tasks["Task Execution"]
  Outbound --> Tasks
  Inbound --> Inventory["Inventory"]
  Outbound --> Inventory
  Layout -->|"Public contract"| Inbound
  Master -->|"Public contract"| Inventory
  Inbound -->|"Domain events"| Integration["Integration"]
  Outbound -->|"Domain events"| Integration
  Mobile -->|"Ordered commands"| Tasks
  Platform -. "Tenant context" .-> Inbound
  Security -. "Policies + audit" .-> Outbound
```

Las flechas representan contratos públicos, no acceso a tablas. Plataforma y Seguridad son capacidades transversales inyectadas y no “god modules”.
